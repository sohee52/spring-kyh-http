# 1. Internet network
## IP (인터넷 프로토콜)

- IP의 역할
    - 지정한 IP 주소에 패킷(Packet)이라는 통신 단위로 데이터 전달
- IP의 한계
    - 비연결성 : 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷이 전송된다.
    - 비신뢰성 : 중간에 패킷이 사라지거나 패킷이 순서대로 오지 않을 수 있다.
    - 프로그램 구분 : 같은 IP를 사용하는 서버에서 통신하는 애플리케이션이 둘 이상일 때 구분하기 어렵다.

## TCP, UDP

- 데이터가 전달되는 과정을 인터넷 프로토콜 스택의 4계층과 함께 설명해 보세요.

  “프로그램이 메시지를 생성하면, 이 메시지는 계층을 따라 아래로 내려가며 각 계층의 역할에 맞게 캡슐화됩니다.”

    1. **애플리케이션 계층**
        - 예: 웹 브라우저, 메신저 앱
        - `SOCKET` 라이브러리를 통해 메시지 전송을 요청
    2. **전송 계층 (TCP/UDP)**
        - 메시지에 TCP/UDP 헤더를 붙여 **프로세스 간 통신**을 구분
            - 프로세스 간 통신을 구분한다. = 같은 IP에서 여러 애플리케이션이 동시에 통신할 때, 포트 번호로 프로세스를 구분해주는 역할을 한다.
        - TCP는 신뢰성(순서 보장, 재전송), UDP는 속도 중심
    3. **인터넷 계층 (IP)**
        - TCP/UDP 세그먼트를 감싸서 **IP 패킷** 생성
        - 송신지/수신지 IP 주소를 붙여 “어느 컴퓨터로 갈지” 지정
    4. **네트워크 인터페이스 계층 (Ethernet 등)**
        - IP 패킷을 다시 **이더넷 프레임**으로 감싸 전송
        - 실제 물리적인 네트워크(유선, 무선)를 통해 흘러감
- IP 패킷 정보
    - 출발지 IP, 목적지 IP, 기타 …
- TCP 패킷 정보
    - 출발지 PORT, 목적지 PORT, 전송 제어, 순서, 검증 정보 …
- TCP 정의와 특징
    - = 전송 제어 프로토콜, Transmission Control Protocol
    - 연결 지향 - TCP 3 way handshake (가상 연결)
        - 연결을 한 다음에 데이터를 보낸다.
    - 데이터 전달 보증
    - 순서 보장
    - 신뢰할 수 있는 프로토콜
    - 현재는 대부분 TCP 사용
- TCP 3 way handshake
    - 1단계 - SYN : 접속 요청
    - 2단계 - SYN + ACK : 접속 요청 + 요청 수락
    - 3단계 - ACK : 요청 수락과 함께 데이터 전송 가능
- UDP 정의와 특징
    - = 사용자 데이터그램 프로토콜, User Datagram Protocol
    - 연결 지향 - TCP 3 way handshake X
    - 데이터 전달 및 순서 보장 X
    - 단순하고 빠름
    - IP + PORT + 체크섬
    - 애플리케이션에서 추가 작업 필요

## PORT

- PORT 의미
    - 같은 IP 내에서 프로세스를 구분해주는 역할
- PORT 숫자 종류
    - 0 ~ 65535 : 할당 가능
    - 0 ~ 1023 : 잘 알려진 포트, 사용하지 않는 것이 좋음
    - FTP : 20, 21
    - TELNET : 23
    - HTTP : 80
    - HTTPS : 443
- IP의 단점
    - IP는 기억하기 어렵다
    - IP는 변경될 수 있다.
- DNS 의미, 사용 방법
    - 도메인 네임 시스템, Domain Name System
    - 도메인 명을 DNS 서버에 등록하면, 도메인을 IP 주소로 변환해서 사용한다.
